PR #338 Explainer：把 release workflow 收进 lingtai-dev-guide

本 PR 将临时的 lingtai-release-workflow 方法论整理为 lingtai-dev-guide 的 nested reference / subskill，方便随 TUI utilities 一起分发，并为 release blog 提供可复用模板。

结论：这是文档/技能路由改动，不改运行时代码逻辑。它新增一个 reference/release-workflow/ 子技能，保留发布授权边界，并把 release blog 模板作为资产纳入 dev guide。

改动范围

新增 subskill

  • tui/internal/preset/skills/lingtai-dev-guide/reference/release-workflow/SKILL.md
  • 覆盖 paired TUI/Portal + kernel release、clean worktrees、validation gates、GitHub/PyPI/Homebrew 边界、website release log/blog。

新增模板资产

  • reference/release-workflow/assets/release-blog-template.md
  • 强制先检查 lingtai-web 过去 release entries 与 GitHub Releases，再填写 release mode、candidate heads、baseline/ranges、严格 delta vs 背景、双语字段、validation/contributors/public-private checks。

更新根路由

  • lingtai-dev-guide/SKILL.md 版本升到 2.3.0。
  • Nested reference catalog、routing table、common examples、layout 都指向新 subskill。

保留旧 compact release guide

  • reference/releasing/SKILL.md 仍是紧凑 release procedure。
  • 新增提示：重大 paired release / website release blog 继续读 reference/release-workflow/SKILL.md。

为什么这样做

  • release workflow 属于 LingTai 开发者方法论，应随 lingtai-dev-guide 一起分发，而不是只留在某个 agent 的本地 custom skill。
  • release blog 生成有固定难点：必须贴合 lingtai-web 既有 releases.ts / ReleaseDetail.astro 和 GitHub Releases 的写法，不能变成自创 review HTML；同时网站结构会变、草稿不等于发布许可、严格 post-tag delta 容易和同一工作窗口背景混淆、中文/英文字段容易不一致。模板把这些变成必填检查项。
  • 保留 progressive disclosure：根 dev guide 仍然短；需要 release 时才进入 reference/release-workflow/。

验证

通过：

  • git diff --check
  • go test ./internal/tui -run TestBuildSkillFolderEntries_DevGuideNestedReferences -v
  • go test ./internal/preset/...
  • 本地 quick check：所有新增/更新 SKILL.md frontmatter 存在，模板资产包含 small-patch | full-release-window | retrospective 等关键字段。

发布边界

注意：本 PR 只把 release workflow/template 纳入 dev guide。它不发布 TUI/kernel 版本，不改网站，不打 tag，不上传 PyPI，也不更新 Homebrew。真正 release 仍需 Jason 明确授权。

Review checklist

  • 确认 reference/release-workflow/SKILL.md 的授权边界足够清楚。
  • 确认 release blog template 的字段是否满足后续网站 release log 需求。
  • 确认 dev-guide 根路由没有变得过长，仍符合 progressive disclosure。
