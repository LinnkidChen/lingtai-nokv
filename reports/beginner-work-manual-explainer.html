LingTai docs PR explainer · prepared 2026-06-04

Beginner-friendly LingTai work manual

This report explains the docs-only change that turns GitHub issue #198 into a maintainable beginner manual plus a standalone animated HTML version.

TL;DR

  • Adds docs/beginner-work-manual.zh.md, a plain-language Chinese beginner manual.
  • Adds docs/beginner-work-manual-stick-figure.zh.html, a self-contained stick-figure animated version.
  • Adds README entry points in README.zh.md, README.md, and README.wen.md.
  • Addresses Lingtai-AI/lingtai#198.

Baseline

Issue #198 asked for a less developer-centric user manual that explains installation, first launch, API/model setup concepts, slash commands, built-in capabilities, daemon/avatar, memory layers, context/molt, and safety/privacy reminders. The current README already frames LingTai as an AI organization and documents installation and architecture, but there was no single beginner-oriented manual linked from the docs path.

What was done

  • Wrote a Markdown manual using beginner-first explanations and source-boundary notes.
  • Included sections requested by the human review pass: how to trigger daemon/avatar, what to do when context explodes, what common slash commands mean, and what soul flow is.
  • Copied the reviewed standalone HTML animation into docs/ with inline CSS/SVG and no remote assets.
  • Linked the manual from localized READMEs so new users can find it without digging through issues.

Validation

  • python3 -m html.parser-equivalent parser check for the standalone HTML and this explainer.
  • git diff --check.
  • Secret/private-path grep for local home-path patterns, sk-..., Bearer ..., and Telegram-token-like patterns in the new docs/report.
  • Manual inspection of README link insertion points and issue #198 requirements.

Risks / decisions

  • The manual is Chinese-first because issue #198 and the reviewed draft are Chinese. English README gets a pointer rather than a full translation.
  • Slash-command descriptions are intentionally marked version-sensitive and defer to the live TUI/README.
  • No code or runtime behavior changes are included.

Next steps

  • Maintainers can decide whether to translate the manual fully into English.
  • If TUI command names change, update the manual alongside README command docs.
  • The HTML animation can be embedded or linked from future website/tutorial pages.
