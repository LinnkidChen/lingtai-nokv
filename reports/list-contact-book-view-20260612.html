Decentralized lingtai-tui list contact-book view

PR-ready explainer for Jason — no centralized address book, live/local agent state only.

Conclusion

Implemented. lingtai-tui list now behaves like a layered contact-book view while preserving the old process-list workflow. It marks main agents directly from each running agent's local .agent.json; it does not create ~/.lingtai-tui/addressbook.jsonl or any global contact database.

Modes

CommandWhat it shows
lingtai-tui list [dir]PID, uptime, role (MAIN/AGENT/HUMAN/?), agent dir name, project.
lingtai-tui list --detailed [dir]Adds public identity/state fields: state, address, agent name, nickname, project path, and agent directory, and public IM handles from each agent-owned `system/mcp_identities/*.json` identity file.
lingtai-tui list --admin [dir]Adds the admin-map summary used as the main-agent判定依据; implies detailed mode.

Decentralization rule

IM handles are intentionally not centralized. The tutorial communication skill now says agents that want to be reachable through Telegram, Feishu, WeChat, or similar channels should publish and maintain those handles in their own durable public profile, project skill, pad, or project context. list helps you find which main agent to ask; it is not the source of truth for IM handles.
