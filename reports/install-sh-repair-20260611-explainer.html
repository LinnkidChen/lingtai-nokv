install.sh source-helper repair

Focused repair for the source-build helper while keeping Homebrew as the primary distribution path.

Conclusion

Scope kept narrow: this patch does not replace Homebrew distribution, does not add release artifacts, and does not change TUI automatic upgrade. It only makes install.sh safer and clearer for people who use the source-build fallback.

What changed

  • Corrected the script header/help text: install.sh is a source-build helper; Homebrew remains primary.
  • Added -h/--help and clearer unknown-flag handling.
  • Added cleanup via trap so temporary build directories are removed on failures.
  • Made invalid --ref fail clearly instead of silently falling back to main.
  • Improved missing dependency hints for common package managers on macOS/Linux.

Preserved behavior

  • Homebrew bin dir is still preferred when brew exists.
  • Non-Homebrew installs still fall back to writable /usr/local/bin or ~/.local/bin.
  • Portal build is still skipped when npm is missing.
  • China-friendly Go/npm mirror auto-detection is unchanged.

Validation

  • bash -n install.sh — passed
  • ./install.sh --help — passed
  • git diff --check — passed

Not run: full source build/install, because that would compile and overwrite local binaries. The change is limited to shell control flow, help text, dependency hints, cleanup, and ref checkout behavior.
