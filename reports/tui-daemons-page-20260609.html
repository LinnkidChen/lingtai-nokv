TUI /daemons page

Read-only daemon run browser for LingTai's terminal UI. The implementation adds a slash command, a Ctrl+T agent switcher, and a two-pane daemon list/detail page.

Command
/daemons opens the new page from the slash palette and generated command list.
Agent switch
ctrl+t opens an agent picker, matching existing TUI conventions from kanban/mailbox/system/library.
Data source
Reads local runtime artifacts only: daemon.json, logs/events.jsonl, history/chat_history.jsonl, and result.txt.
Safety
No lifecycle actions, deletes, or daemon mutations. The selected daemon detail renders the full task, full chat_history turns, full event/tool JSON lines, and full result text read-only.

What users see

RegionContent
HeaderDaemons — <agent> showing the selected agent.
Left paneNewest-first daemon runs, each with handle (em-7), state, and task snippet.
Right paneSelected run metadata, task, full event/tool records, full interactions, and full result text.
FooterNavigation hints: select with arrows/jk, refresh with r, switch agent with Ctrl+T, Esc back.

Limitations / follow-ups

This PR intentionally keeps the first version local and read-only, but the detail view is no longer summary-only: it displays the full task, interaction process, event/tool records, and result text for the selected daemon. Possible future iterations: richer filters by state/backend, search within task/result text, direct jump to raw files, or live daemon process controls if the product direction wants mutation from this page.
